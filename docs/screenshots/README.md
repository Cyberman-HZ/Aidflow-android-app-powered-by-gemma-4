# Screenshots — drop your images here

Save your phone screenshots in this folder using the filenames below. The main
[README.md](../../README.md) already references these paths, so as soon as a
file with the right name lands here it shows up in the README on GitHub.

Recommended filenames (PNG, portrait, ideally 1080 × 2400 from a Pixel 7a):

| Filename | What to capture |
|---|---|
| `01-home.png` | Home screen showing all four feature cards |
| `02-family-intake.png` | Family Intake screen — Speak mode with the description text field |
| `03-family-extracted.png` | Family Intake after extraction — the editable draft card with the priority chip |
| `04-family-captured.png` | Family Intake "Captured this session" list with 2–3 families |
| `05-items.png` | Identify Items screen with one or two items already identified |
| `06-scan-translated.png` | Translate a Document with Original / Cleaned / Translated cards |
| `07-translate-voice.png` | Voice/text translate mid-conversation (with a translation visible) |
| `08-export-sheet.png` | Export bottom sheet showing the scope chips + format buttons |
| `09-doc-scanner.png` | Google document scanner UI (camera viewfinder, ideally with the green edge-detection outline visible) |
| `10-setup.png` | Model setup screen during first-run download or engine load |

If you only have some of these, that's fine — the README will just show the
ones that exist. You can also rename existing screenshots to fit; do not
worry about exact resolution.

## How to take Android screenshots

- Power + Volume Down at the same time, or
- Pull down the quick-settings panel and tap the Screenshot tile (on most
  modern Android versions).

They land in `~/Pictures/Screenshots/` on your phone. Copy them over (USB,
Google Drive, etc.) and drop them into this folder.

## How to crop / annotate (optional)

If you want to add arrows or callouts, [Carbon](https://carbon.now.sh) is
overkill for screenshots — just use the built-in Markup tool that opens after
you take a screenshot on your phone. Save and re-export the annotated PNG with
the right filename.
